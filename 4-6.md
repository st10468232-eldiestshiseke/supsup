Explain the importance of abstraction levels in network architecture design.

Answer: Abstraction simplifies complexity by hiding lower-level details, allowing engineers to focus on specific layers (e.g., OSI model). It enables modularity, interoperability, and easier troubleshooting. For example, TCP/IP abstracts physical transmission from application protocols.

Resource pooling (e.g., VMs sharing a physical server).
Isolation (e.g., containers for microservices).
Flexibility (e.g., rapid scaling with NFV/SDN).
Cost savings (reduced hardware dependency).

Explain the purpose and benefits of network segmentation.

Answer:

Purpose: Divide a network into smaller, isolated segments to:

Improve security (limit lateral movement of threats).
Enhance performance (reduce broadcast traffic).
Simplify management (e.g., apply policies per segment).

Benefits:

Isolation: Contain breaches (e.g., separate VLAN for PCI-DSS compliance).
Traffic Control: Prioritize critical traffic (e.g., VoIP).
Compliance: Meet regulatory requirements (e.g., GDPR, HIPAA).





Analyse how VLANs improve network security and performance.

Answer:

Security:

Isolate sensitive traffic (e.g., HR VLAN separate from guest VLAN).
Reduce attack surface (e.g., limit broadcast domains).

Performance:

Reduce broadcast storms (e.g., ARP requests confined to VLAN).
Improve bandwidth utilization (e.g., prioritize VLANs with QoS).

Example: A university uses VLANs to separate student, faculty, and admin traffic.




Explain how trunking supports communication between multiple VLANs.

Answer:

Trunking (802.1Q) allows a single physical link to carry multiple VLANs by adding a VLAN tag (4-byte header) to Ethernet frames.
How it works:

Switch adds a VLAN ID (e.g., VLAN 10 = 0x0A) to frames.
Frames travel over the trunk link.
Receiving switch removes the tag and forwards to the correct VLAN.

Example: A trunk link between two switches carries traffic for VLANs 10, 20, and 30.




Apply subnetting calculations to create efficient network addressing plans.

Answer:

Steps:

Determine host requirements (e.g., 50 hosts per subnet → need 56 IPs [2^6 - 2]).
Choose subnet mask (e.g., /26 for 64 IPs).
Calculate network/broadcast addresses (e.g., 192.168.1.0/26: Network = 192.168.1.0, Broadcast = 192.168.1.63).
Assign subnets (e.g., 192.168.1.64/26 for next subnet).

Example:

Network: 192.168.0.0/24
Subnet Mask: 255.255.255.192 (/26)
Subnets: 192.168.0.0, 192.168.0.64, 192.168.0.128, 192.168.0.192
Usable IPs per Subnet: 62 (64 - 2 for network/broadcast).





Analyse the importance of CIDR and VLSM in modern networking.

Answer:

CIDR (Classless Inter-Domain Routing):

Replaces classful addressing (A/B/C) with flexible subnet masks (e.g., /24, /26).
Enables route aggregation (e.g., 192.168.0.0/24 instead of 256 /32 routes).

VLSM (Variable Length Subnet Masking):

Allows subnets of different sizes within a network (e.g., /24 for HQ, /28 for branches).
Improves IP address efficiency (e.g., avoid wasting IPs in small subnets).

Example: An ISP uses CIDR to allocate /20 blocks to customers and VLSM to further divide them.




Evaluate the operational impact of poor subnetting design.

Answer:

IP Exhaustion: Running out of usable IPs due to inefficient allocation.
Routing Overhead: Large routing tables slow down routers.
Broadcast Storms: Too many hosts in a subnet increase broadcast traffic.
Security Risks: Flat networks allow lateral movement for attackers.
Example: A /24 subnet for 200 hosts causes IP shortages and performance issues.




Discuss how segmented networks improve traffic management and reduce broadcast domains.

Answer:

Traffic Management:

Prioritization: QoS policies can prioritize critical traffic (e.g., VoIP over file transfers).
Isolation: Separate VLANs for voice/video/data reduce congestion.

Broadcast Domains:

Broadcasts (e.g., ARP requests) are confined to the VLAN, reducing unnecessary traffic.
Example: A broadcast from VLAN 10 won’t reach VLAN 20, improving efficiency.

Forward packets between networks using routing tables.

Evaluate the impact of scalability and geographic coverage on WAN design decisions.

Hierarchical Design: Core-distribution-access layers.

Geographic Coverage:

Latency: Longer distances increase delay (e.g., satellite ~500ms).
Solutions:

CDNs: Cache content closer to users.
Anycast: Route requests to nearest server (e.g., DNS root servers).

Explain the principles of wireless transmission in WLAN environments.

Answer:

Principles:

Radio Frequency (RF): Uses 2.4 GHz or 5 GHz bands (IEEE 802.11).
Modulation: Encodes data onto RF signals (e.g., OFDM in 802.11n/ac).
Spread Spectrum: Reduces interference (e.g., DSSS, FHSS).
CSMA/CA: "Listen before talk" to avoid collisions (unlike Ethernet’s CSMA/CD).

Key Concepts:

SSID: Network name (e.g., "CoffeeShop_WiFi").
BSSID: MAC address of the AP.
Channels: Non-overlapping (e.g., 1, 6, 11 in 2.4 GHz).

Evaluate the importance of channel planning in wireless deployments.

Answer:

Why It Matters:

Avoids co-channel interference (e.g., two APs on channel 6).
Maximizes throughput by reducing collisions.

Best Practices:

5 GHz: More channels available (e.g., 36, 40, 44, 48).
Auto-Selection: Use tools like Wi-Fi analyzers to pick the least congested channel.

Example: In an office with 3 APs, assign channels 1, 6, 11 to avoid overlap.

Analyse the impact of RF interference on wireless network performance.

Answer:

Sources of Interference:

Co-Channel: Overlapping Wi-Fi channels (e.g., two APs on channel 6).
Adjacent-Channel: Nearby channels (e.g., channel 5 and 6).
Non-Wi-Fi: Microwaves, Bluetooth, cordless phones, radar.

Impact:

Lower Throughput: Retransmissions due to collisions.
Higher Latency: Delays from waiting for clear channels.
Dropped Connections: Clients disconnect if signal is too weak.

Mitigation:

Use 5 GHz (less crowded than 2.4 GHz).
Channel Planning: Avoid overlapping channels.
DFS Channels: Use radar-free channels in 5 GHz (e.g., 36-48).

🔒 Learning Unit 6: Risk Management, Network Security, Performance and Recovery


Explain the importance of confidentiality, integrity, and availability (CIA) in network security.

Answer:

Confidentiality:

Definition: Ensure data is accessible only to authorized users.
Methods: Encryption (e.g., AES, TLS), access controls (e.g., ACLs).
Example: Encrypting emails with PGP.

Integrity:

Definition: Ensure data is unaltered and trustworthy.
Methods: Hashing (e.g., SHA-256), digital signatures, checksums.
Example: MD5 checksums for file verification.

Availability:

Definition: Ensure systems/data are accessible when needed.
Methods: Redundancy (e.g., load balancers), DDoS protection, backups.
Example: AWS Auto Scaling for web servers.





Analyse how defence-in-depth strategies reduce organisational security risks.

Answer:

Definition: Layered security approach (no single point of failure).
Layers:

Physical: Biometric locks, surveillance.
Network: Firewalls, IDS/IPS.
Endpoint: Antivirus, EDR (Endpoint Detection and Response).
Application: WAF (Web Application Firewall), input validation.
Data: Encryption (at rest/in transit).
User: MFA, security training.

Example: A bank uses:

Firewall (network layer).
SIEM (monitoring layer).
MFA (user layer).
Disk Encryption (data layer).
